ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Web-сервис «Генератор дипломных идей» (DiplomaSpark)
1. Общая концепция
Сервис помогает студентам генерировать оригинальные, актуальные и выполнимые темы для выпускных квалификационных работ (дипломов). Студент заполняет короткую анкету (специальность, интересы, доступные ресурсы, сроки), нажимает кнопку – и получает 3-5 идей с детальным описанием: актуальность, новизна, структура, методы, результаты, сложность.

Уникальность сервиса:

Генерация происходит через AI (LLM), но владелец сервиса не платит за API заранее. Он вставляет свои ключи от OpenAI/Anthropic/etc в админ-панели. Сервис использует их для запросов.

Никакой базы данных – всё в файлах. Это упрощает хостинг и бэкапы.

Красивый, адаптивный интерфейс (студенты заходят с телефонов, планшетов, ноутбуков).

2. Пользователи и роли
Администратор (ты) – один человек. Доступ к админ-панели по логину/паролю (зашиты в коде или в переменных окружения).

Студенты – неограниченное число. Не требуют регистрации. Могут генерировать идеи (с ограничением по IP или без – настраивается в админке).

3. Функциональные требования
3.1. Админ-панель
Доступна по /admin. Защищена базовой аутентификацией (логин: admin, пароль: генерируется при первом запуске и выводится в консоль, потом можно сменить через файл config.json).

Содержит разделы:

3.1.1. Управление API-ключами

Список провайдеров: OpenAI, Anthropic, Google Gemini, Cohere, Mistral, (можно добавить любые через интерфейс).

Для каждого – поле для ввода API-ключа, кнопка «Сохранить», кнопка «Проверить ключ» (тестовый запрос к API, например, к моделям list или простой completion). Результат проверки: «Ключ рабочий» или «Ошибка: ...».

Можно включить/выключить использование конкретного провайдера (checkbox). Сервис будет использовать первый доступный рабочий ключ по приоритету (порядок перетаскиванием).

После сохранения ключей сервис перезагружает конфиг без перезапуска всего приложения.

3.1.2. Настройки генерации

Поля (сохраняются в settings.json):

default_model – какая модель используется по умолчанию (gpt-4-turbo, claude-3-opus и т.д.).

max_topics_per_request – сколько максимум тем генерировать (по умолчанию 5).

temperature – креативность (0.2 – консервативно, 0.8 – креативно).

timeout_seconds – таймаут на запрос к API (30 сек).

rate_limit_per_ip_per_hour – сколько запросов от одного IP в час (по умолчанию 5). Это чтобы студенты не злоупотребляли.

save_all_requests – сохранять ли каждый запрос студента в файл (true/false). Полезно для анализа.

enable_pdf_export – разрешить скачивание в PDF.

Кнопка «Сохранить настройки».

3.1.3. Статистика (из файлов)

Общее количество генераций (сколько раз студенты нажали «Сгенерировать»).

Список самых популярных специальностей (топ-10).

Список самых частых ключевых слов в интересах.

График генераций по дням (простой bar chart на Chart.js).

Лог ошибок API (таймауты, невалидные ключи) – таблица с временем и текстом ошибки.

Кнопка «Очистить логи» и «Скачать логи в CSV».

3.1.4. Управление промптами (шаблонами)

Владелец может редактировать текст промпта, который отправляется в AI. По умолчанию он уже настроен (см. раздел 4.2). Но можно изменить через текстовое поле в админке.

Можно создать несколько версий промптов (A/B тест) – но для простоты MVP оставим один редактируемый промпт.

3.1.5. Системная информация

Версия Python, список установленных пакетов.

Использование диска (сколько места занято логами/кэшем).

Кнопка «Очистить кэш» (удалить временные файлы, если есть).

3.2. Студенческий интерфейс
Главная страница / – красивая форма с анимацией.

Поля формы (все обязательные, кроме галочек):

Специальность – выпадающий список (можно выбрать из предустановленных: «Прикладная информатика», «Экономика», «Психология», «Дизайн», «Биология», «Машиностроение», «Филология» и т.д. + пункт «Другое» с текстовым полем). Список специальностей редактируется в админ-панели (простым текстовым файлом specialties.txt).

Ваши интересы – текстовое поле (например: «компьютерное зрение, Python, Telegram-боты»). Подсказка: «Напишите через запятую, что вам нравится».

Доступные ресурсы – чекбоксы:

Доступ к реальным данным (опросы, статистика предприятия)

Лабораторное оборудование (датчики, 3D-принтер)

Бесплатный хостинг / сервер

Навыки программирования (Python, R, JS)

Другое (текстовое поле)

Срок до защиты – выпадающий список: «1 месяц», «2-3 месяца», «4-6 месяцев», «больше 6 месяцев».

Желаемый тип работы – радиокнопки: «Теоретическая», «Экспериментальная/лабораторная», «Разработка прототипа (софт/сайт)», «Анализ данных», «Смешанный».

Уровень – бакалавр / магистр.

Хотите использовать AI в работе? – да/нет (если да, то AI-темы будут более технологичными).

Кнопка «Сгенерировать идеи». Во время генерации – анимация загрузки (спиннер), текст «Анализируем ваши данные, это займёт 10-20 секунд».

После генерации появляется 3-5 карточек (в зависимости от настроек). Каждая карточка содержит:

Название темы (жирный шрифт)

Актуальность (2-3 предложения)

Научная/прикладная новизна

Структура (список глав: Глава 1, Глава 2...)

Методы исследования (например: контент-анализ, эксперимент, нейросетевое моделирование)

Ожидаемый результат (что студент создаст: прототип, база данных, статья, набор рекомендаций)

Необходимые ресурсы (какое ПО, данные, время)

Сложность (легко / средне / сложно) и примерный объём страниц

Кнопки:

«Сохранить в избранное» (сохраняется в localStorage браузера студента, не на сервер)

«Скачать как TXT» (генерирует .txt файл)

«Скачать как PDF» (если включено в настройках)

«Плохая идея, сгенерировать ещё» – отправляет тот же запрос, но с просьбой переформулировать (используется тот же промпт с флагом regenerate=true).

Внизу страницы – кнопка «Новый запрос» (очистить форму и начать заново).

3.3. Дополнительные фичи (необязательные, но крутые)
История генераций для студента (сохраняется в localStorage браузера, чтобы студент мог вернуться к предыдущим идеям без сервера).

Режим «Анти-паника» – отдельная кнопка на главной: «У меня вообще нет идей, спасите!» – форма сокращается до одного поля «Что вы умеете?» и генерируется одна тема на основе любых доступных ключей.

Кнопка «Усилить тему» – после генерации студент может выбрать любую тему и нажать «Усилить», тогда AI добавит более сложные элементы (сравнение нескольких моделей, внедрение в реальную организацию и т.п.).

5. Логика работы AI-генерации (ядро сервиса)
5.1. Модуль ai_engine.py
Класс AIEngine:

При инициализации загружает api_keys.json и settings.json.

Метод get_available_provider(): перебирает provider_order, проверяет enabled и наличие непустого ключа, возвращает имя провайдера и клиент.

Метод generate_topics(student_data):

Считывает системный промпт из system_prompt.txt.
Формирует user-промпт на основе полей студента.
Вызывает соответствующий API (отправляет запрос).
Парсит ответ (ожидается JSON-подобная структура или просто текст, разбитый на блоки). Рекомендуется просить AI вернуть JSON строго по схеме, чтобы легко разобрать.
Обрабатывает ошибки (таймаут, невалидный ключ, rate limit) и логирует в errors.log.
Возвращает список тем (как список словарей).
Пример промпта (user part):

text
Студент специальности: {specialty}
Интересы: {interests}
Ресурсы: {resources}
Срок до защиты: {deadline}
Тип работы: {work_type}
Уровень: {level}
Использовать AI: {use_ai}

Сгенерируй {max_topics} уникальных, оригинальных, выполнимых тем для дипломной работы. Для каждой темы предоставь JSON-объект со следующими полями: title, relevance, novelty, structure (массив строк глав), methods (строка), expected_result, required_resources, difficulty (easy/medium/hard), pages_approx.

Верни список из {max_topics} таких объектов.
5.2. Парсинг ответа
Используем json.loads() после очистки от markdown-обертки (иногда AI возвращает json ...). Если парсинг не удался – записываем ошибку и возвращаем fallback (3 темы-заглушки с текстом «Ошибка генерации, попробуйте ещё раз»).

6. Админ-панель: детали реализации
6.1. Защита доступа
При первом запуске генерируется случайный пароль (например, secrets.token_urlsafe(12)), выводится в консоль и сохраняется в .env или в отдельный файл admin_credentials.json. Владелец может сменить пароль через интерфейс.

Используем fastapi.security.HTTPBasic или простую форму с сессией (через session в FastAPI + itsdangerous). Для простоты можно использовать fastapi-login или реализовать проверку через промежуточное ПО (middleware) на пути /admin/*.

6.2. Роуты админки
GET /admin – редирект на /admin/login или сразу дашборд (если уже залогинен).

POST /admin/login – проверка пароля, установка cookie/сессии.

GET /admin/dashboard – основная панель с плитками (управление ключами, настройки, статистика).

GET /admin/keys – страница управления API-ключами (см. п.3.1.1).

POST /admin/keys/save – сохраняет ключи в api_keys.json.

GET /admin/settings – страница настроек.

POST /admin/settings/save – сохраняет settings.json.

GET /admin/stats – статистика.

GET /admin/prompts – редактирование системного промпта.

POST /admin/prompts/save – сохраняет system_prompt.txt.

POST /admin/clear_logs – очищает логи.

6.3. Интерфейс админки
Используем простые HTML-шаблоны с использованием TailwindCSS для стилей. Навигация – сайдбар слева. Графики для статистики – Chart.js (подгружается из CDN). Таблицы – сортируемые.

7. Студенческий интерфейс: детали
7.1. Роуты
GET / – главная форма.

POST /generate – принимает JSON с данными формы, вызывает ai_engine.generate_topics(), возвращает JSON со списком тем. Также логирует запрос (если save_all_requests true) и обновляет статистику в stats.json.

GET /download_pdf – принимает тему (в query параметре как JSON) и возвращает PDF-файл. Генерируется на лету с помощью reportlab.

7.2. Frontend
Используем Fetch API для отправки формы без перезагрузки страницы.

Анимация загрузки – спиннер из CSS.

Вывод карточек – динамически добавляем в DOM.

Сохранение в избранное – через localStorage (ключ favorites).

Ограничение по IP реализовано на бэкенде: в middleware проверяем request.client.host и смотрим rate_limit_per_ip_per_hour в settings.json. Храним счётчики в словаре ip_requests (сброс по cron каждый час или просто проверяем время последнего запроса). Для простоты можно хранить в файле ip_limits.json с временными метками.

Сайт готов к использованию студентами.

10. Обработка ошибок и надёжность
Если API-ключ невалидный или истёк – админ-панель покажет ошибку. Студенту в таком случае будет показано сообщение: «Сервис временно недоступен, администратор уже чинит. Попробуйте позже».

При rate limit от OpenAI – автоматически переключиться на следующего провайдера (если есть). Если ни одного нет – ошибка.

Все ошибки API логируются в errors.log с полным traceback (для отладки).

Таймаут запроса – не более 30 секунд, иначе отмена и возврат fallback.

11. Расширяемость
Легко добавить нового провайдера: дописать в ai_engine.py метод _call_provider_x(), добавить в provider_order и в админ-панель новый блок ключей.

Легко изменить промпт через админку – не нужно перезапускать сервер.

Можно добавить платную подписку для студентов, но по ТЗ это не требуется.

app/ai_engine.py (упрощённая версия, поддерживает OpenAI, легко расширяется)
python
import json
import asyncio
from openai import AsyncOpenAI
from app.config_manager import load_api_keys, load_settings, load_system_prompt

class AIEngine:
    def __init__(self):
        self.settings = load_settings()
        self.keys = load_api_keys()
        self.system_prompt = load_system_prompt()
    
    async def generate_topics(self, student_data: dict):
        # Выбираем первого доступного провайдера
        provider = None
        api_key = None
        for p in self.settings["provider_order"]:
            if self.keys.get(p, {}).get("enabled") and self.keys[p].get("key"):
                provider = p
                api_key = self.keys[p]["key"]
                break
        if not provider:
            raise Exception("Нет доступных API-ключей. Зайдите в админ-панель и добавьте ключи.")
        
        # Формируем промпт
        user_prompt = f"""
Студент специальности: {student_data.get('specialty')}
Интересы: {student_data.get('interests')}
Ресурсы: {student_data.get('resources')}
Срок до защиты: {student_data.get('deadline')}
Тип работы: {student_data.get('work_type')}
Уровень: {student_data.get('level')}
Использовать AI: {student_data.get('use_ai')}

Сгенерируй {self.settings['max_topics_per_request']} уникальных, выполнимых тем для диплома.
Для каждой темы верни JSON-объект с полями:
title, relevance, novelty, structure (массив строк), methods, expected_result, required_resources, difficulty (easy/medium/hard), pages_approx.
Верни список таких объектов.
"""
        if provider == "openai":
            client = AsyncOpenAI(api_key=api_key)
            response = await client.chat.completions.create(
                model=self.settings["default_model"],
                messages=[
                    {"role": "system", "content": self.system_prompt},
                    {"role": "user", "content": user_prompt}
                ],
                temperature=self.settings["temperature"],
                timeout=self.settings["timeout_seconds"]
            )
            content = response.choices[0].message.content
            # Парсим JSON
            content = content.strip().removeprefix("```json").removesuffix("```").strip()
            topics = json.loads(content)
            return topics[:self.settings["max_topics_per_request"]]
        else:
            raise Exception(f"Провайдер {provider} временно не реализован, используйте OpenAI")

@router.post("/api/generate")
async def generate(data: StudentInput, request: Request):
    # Rate limit по IP (упрощённо)
    client_ip = request.client.host
    settings = load_settings()
    # Можно реализовать простой словарь в памяти (для MVP)

        # Логируем запрос

                "topics_count": len(topics)
            }
            log_file = LOGS_DIR / "requests.log"
